---
title: "Health check endpoint — GET /api/v1/healthcheck"
description: "Verify that the LMS Backend server and its MongoDB database connection are healthy. Returns server uptime, memory usage, and database ready-state."
---

The health check endpoint gives you a single, unauthenticated call to confirm that the API server is reachable and that the underlying MongoDB database is connected. It is the first endpoint to call when troubleshooting connectivity or integrating an uptime monitor. The response HTTP status reflects the overall health — `200` when both the server and database are healthy, `503` when the database is unreachable, and `500` for unexpected internal errors.

## Endpoint

<ParamField path="method" type="string">
  `GET`
</ParamField>

```
GET /api/v1/healthcheck
```

**Auth required:** No — this endpoint is public.

---

## Request

No request body, query parameters, or headers are required.

<CodeGroup>

```bash cURL
curl --request GET \
  --url http://localhost:4000/api/v1/healthcheck
```

</CodeGroup>

---

## Response

### 200 — Healthy

Returned when the server is running and the database connection is in the `connected` ready-state.

<ResponseField name="status" type="string" required>
  Overall system status. Value is `"OK"` when healthy.
</ResponseField>

<ResponseField name="timeStamp" type="string" required>
  ISO 8601 timestamp of when the check was performed.
</ResponseField>

<ResponseField name="services" type="object" required>
  Container for individual service statuses.

  <Expandable title="services properties">
    <ResponseField name="databases" type="object">
      MongoDB database health.

      <Expandable title="databases properties">
        <ResponseField name="status" type="string">
          `"healthy"` when connected, `"unhealthy"` otherwise.
        </ResponseField>
        <ResponseField name="details" type="object">
          Raw connection details from Mongoose.

          <Expandable title="details properties">
            <ResponseField name="isConnected" type="boolean">
              Whether the database has an active connection.
            </ResponseField>
            <ResponseField name="readyState" type="string">
              Human-readable connection state: `"disconnected"`, `"connected"`, `"connecting..."`, `"disconnecting"`, or `"unknown"`.
            </ResponseField>
            <ResponseField name="host" type="string">
              The hostname of the MongoDB server as reported by Mongoose.
            </ResponseField>
            <ResponseField name="name" type="string">
              The name of the connected database.
            </ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="server" type="object">
      Node.js process health.

      <Expandable title="server properties">
        <ResponseField name="status" type="string">
          Always `"healthy"` when the process responds. Value is `"healthy"`.
        </ResponseField>
        <ResponseField name="uptime" type="number">
          Number of seconds the Node.js process has been running.
        </ResponseField>
        <ResponseField name="memoryUsage" type="object">
          Node.js `process.memoryUsage()` output — includes `rss`, `heapTotal`, `heapUsed`, `external`, and `arrayBuffers` in bytes.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

```json Example 200 response
{
  "status": "OK",
  "timeStamp": "2024-11-15T10:22:04.321Z",
  "services": {
    "databases": {
      "status": "healthy",
      "details": {
        "isConnected": true,
        "readyState": "connected"
      }
    },
    "server": {
      "status": "healthy",
      "uptime": 3842.17,
      "memoryUsage": {
        "rss": 62423040,
        "heapTotal": 29933568,
        "heapUsed": 23411952,
        "external": 2084281,
        "arrayBuffers": 43082
      }
    }
  }
}
```

### 503 — Database unhealthy

Returned when MongoDB is unreachable or in a non-connected state. The response body has the same shape as the `200` response, but `services.databases.status` is `"unhealthy"` and `services.databases.details.readyState` reflects the actual Mongoose ready-state code.

### 500 — Internal error

Returned when the health check itself throws an unexpected exception.

<ResponseField name="status" type="string">
  Value is `"ERROR"`.
</ResponseField>

<ResponseField name="timeStamp" type="string">
  ISO 8601 timestamp of when the error occurred.
</ResponseField>

<ResponseField name="error" type="string">
  The error message from the thrown exception.
</ResponseField>

---

<Tip>
  Use this endpoint in your container orchestration health probes (Kubernetes `livenessProbe` / `readinessProbe`) or uptime monitoring services. A `200` response means the system is ready to accept traffic.
</Tip>
