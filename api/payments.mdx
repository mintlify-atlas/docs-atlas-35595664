---
title: "Payment endpoints — Stripe checkout and purchases"
description: "Initiate Stripe Checkout sessions for course purchases, handle Stripe webhook events to confirm payments, and query purchase status or list completed purchases."
---

The LMS Backend uses Stripe Checkout to process course purchases. The flow works in three steps: your client calls the create-checkout-session endpoint to get a Stripe session URL, the student completes payment on Stripe's hosted page, and Stripe calls the webhook endpoint to confirm the charge. The `CoursePurchase` document starts in `pending` status and is updated to `completed` or `failed` based on the Stripe `charge.succeeded` or `charge.failed` event.

<Info>
  All payment endpoints are prefixed with `/api/v1/payments`. Set `STRIPE_SECRET_KEY` and `STRIPE_WEBHOOK_SECRET` in your environment before using these endpoints. See [Environment Setup](/environment-setup) for the full variable list.
</Info>

---

## POST /api/v1/payments/create-checkout-session

Create a Stripe Checkout session for a course purchase. The server looks up the course price, creates the Stripe session in `payment` mode, and records a `CoursePurchase` document with `status: "pending"`. The Stripe session URL is returned in the response for your client to redirect the student to.

**Auth required:** Yes

### Request body

<ParamField body="courseId" type="string" required>
  MongoDB ObjectId of the course the student wants to purchase.
</ParamField>

### Response — 201

<ResponseField name="success" type="boolean">
  `true` on success.
</ResponseField>

<ResponseField name="message" type="string">
  `"Stripe checkout initiated successfully."`
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="data properties">
    <ResponseField name="session" type="object">
      The full Stripe Checkout Session object. Key fields:

      <Expandable title="session fields">
        <ResponseField name="id" type="string">Stripe session ID (used as `paymentId` in the `CoursePurchase` record).</ResponseField>
        <ResponseField name="url" type="string">Stripe-hosted checkout URL. Redirect your student to this URL to complete payment.</ResponseField>
        <ResponseField name="status" type="string">Stripe session status, e.g. `"open"`.</ResponseField>
        <ResponseField name="mode" type="string">`"payment"` for one-time purchases.</ResponseField>
        <ResponseField name="success_url" type="string">URL Stripe redirects to after successful payment.</ResponseField>
        <ResponseField name="cancel_url" type="string">URL Stripe redirects to if the student cancels.</ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="coursePurchase" type="object">
      The pending purchase record created in the database.

      <Expandable title="coursePurchase fields">
        <ResponseField name="_id" type="string">MongoDB document ID.</ResponseField>
        <ResponseField name="course" type="string">ObjectId of the purchased course.</ResponseField>
        <ResponseField name="user" type="string">ObjectId of the purchasing user.</ResponseField>
        <ResponseField name="amount" type="number">Course price in USD at the time of purchase.</ResponseField>
        <ResponseField name="currency" type="string">`"USD"`</ResponseField>
        <ResponseField name="paymentMethod" type="string">`"stripe"`</ResponseField>
        <ResponseField name="paymentId" type="string">The Stripe session ID, used to match the webhook event.</ResponseField>
        <ResponseField name="status" type="string">`"pending"` — updated to `"completed"` or `"failed"` via the webhook.</ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="course" type="object">
      Summary of the course being purchased.

      <Expandable title="course summary fields">
        <ResponseField name="title" type="string">Course title.</ResponseField>
        <ResponseField name="description" type="string">Course description.</ResponseField>
        <ResponseField name="amount" type="number">Course price in USD.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<CodeGroup>

```bash cURL
curl --request POST \
  --url http://localhost:4000/api/v1/payments/create-checkout-session \
  --cookie cookies.txt \
  --header 'Content-Type: application/json' \
  --data '{"courseId": "64f1a2b3c4d5e6f7a8b9c0d1"}'
```

</CodeGroup>

### Errors

| Status | Condition |
|--------|-----------|
| `401` | Missing or invalid auth cookie |
| `404` | Course not found |
| `500` | Stripe session creation failed |

## POST /api/v1/payments/webhook

Receive and process Stripe webhook events. This endpoint must receive the **raw** request body (not JSON-parsed) so that Stripe's signature verification can succeed. The route uses `express.raw({ type: 'application/json' })` middleware for this reason.

**Auth required:** No — secured by Stripe signature verification using `STRIPE_WEBHOOK_SECRET`.

<Warning>
  Do not send this endpoint a regular JSON body — the raw bytes are required for the `stripe-signature` header verification. Any proxy or middleware that parses the body before it reaches this route will cause signature verification to fail with a `400` error.
</Warning>

### Headers

<ParamField header="stripe-signature" type="string" required>
  The `Stripe-Signature` header set by Stripe on every webhook delivery. Used to verify the event payload has not been tampered with.
</ParamField>

### Handled event types

<AccordionGroup>
  <Accordion title="charge.succeeded">
    Fired when a payment charge is successfully captured. The handler finds the matching `CoursePurchase` record by `paymentId` (the charge ID) and updates its `status` to `"completed"`.
  </Accordion>

  <Accordion title="charge.failed">
    Fired when a payment charge fails. The handler finds the matching `CoursePurchase` record by `paymentId` and updates its `status` to `"failed"`.
  </Accordion>
</AccordionGroup>

All other event types are acknowledged but not processed.

### Response — 200

```json
{
  "received": true
}
```

A `200` response is always returned once the event is acknowledged, even for unhandled event types. This tells Stripe not to retry delivery.

<CodeGroup>

```bash cURL — local testing with Stripe CLI
# Forward events to your local server using the Stripe CLI:
stripe listen --forward-to http://localhost:4000/api/v1/payments/webhook

# Trigger a test charge.succeeded event:
stripe trigger charge.succeeded
```

</CodeGroup>

### Errors

| Status | Condition |
|--------|-----------|
| `400` | Stripe signature verification failed (wrong secret, tampered body, or body was pre-parsed) |

## GET /api/v1/payments/courses/:courseId/purchase-status

Check whether the authenticated user has purchased a specific course and what the current status of that purchase is.

**Auth required:** Yes

### Path parameters

<ParamField path="courseId" type="string" required>
  MongoDB ObjectId of the course to check.
</ParamField>

### Response — 200

<ResponseField name="status" type="string">
  `"success"`
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="data properties">
    <ResponseField name="course" type="object">
      Course summary with `_id`, `title`, `description`, and `price`.
    </ResponseField>

    <ResponseField name="purchaseStatus" type="string">
      The purchase status for this user and course. Possible values:

      | Value | Meaning |
      |-------|---------|
      | `"pending"` | A checkout session was started but payment has not been confirmed yet. |
      | `"completed"` | Payment was confirmed via webhook. The user has full access. |
      | `"failed"` | The charge failed. The user does not have access. |
      | `"not purchased"` | No purchase record exists for this user and course. |
    </ResponseField>
  </Expandable>
</ResponseField>

<CodeGroup>

```bash cURL
curl --request GET \
  --url http://localhost:4000/api/v1/payments/courses/64f1a2b3c4d5e6f7a8b9c0d1/purchase-status \
  --cookie cookies.txt
```

</CodeGroup>

```json Example 200 response — completed purchase
{
  "status": "success",
  "data": {
    "course": {
      "_id": "64f1a2b3c4d5e6f7a8b9c0d1",
      "title": "Introduction to Node.js",
      "description": "A complete beginner course covering Express, MongoDB, and authentication.",
      "price": 49
    },
    "purchaseStatus": "completed"
  }
}
```

### Errors

| Status | Condition |
|--------|-----------|
| `401` | Missing or invalid auth cookie |
| `404` | Course not found |

## GET /api/v1/payments/purchased-courses

Retrieve all courses the authenticated user has purchased with a `"completed"` status. Results are sorted by most recently purchased and include the course title, description, and price.

**Auth required:** Yes

### Response — 200

<ResponseField name="status" type="string">
  `"success"`
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="data properties">
    <ResponseField name="purchase" type="array">
      Array of completed purchase records, each containing:

      <Expandable title="purchase record fields">
        <ResponseField name="_id" type="string">MongoDB document ID of the purchase record.</ResponseField>
        <ResponseField name="course" type="object">
          Populated course with `_id`, `title`, `description`, and `price`.
        </ResponseField>
        <ResponseField name="amount" type="number">Amount paid in USD.</ResponseField>
        <ResponseField name="currency" type="string">Currency code, e.g. `"USD"`.</ResponseField>
        <ResponseField name="paymentMethod" type="string">`"stripe"`</ResponseField>
        <ResponseField name="paymentId" type="string">Stripe session ID.</ResponseField>
        <ResponseField name="createdAt" type="string">ISO 8601 purchase timestamp.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<CodeGroup>

```bash cURL
curl --request GET \
  --url http://localhost:4000/api/v1/payments/purchased-courses \
  --cookie cookies.txt
```

</CodeGroup>

```json Example 200 response

  }

```
